# coMGan-ai 主要功能页面小任务拆分清单

本文档基于《主要功能页面开发明细表.md》继续拆分，目标是把 coMGan-ai 的 11 步创作中心开发拆成尽可能小、可独立完成、可独立验证、可独立提交的小任务。

## 执行规则

| 规则 | 说明 |
| --- | --- |
| 每次只完成一个小任务 | 后续开发默认一次只做一个任务编号，例如 `G-001`、`S03-004`、`API-012`。除非用户明确说“连续做多个”，否则不能顺手扩展到下一项。 |
| 单任务必须可验收 | 每个任务都要有明确交付物和验收点，避免“做一部分页面”这种模糊范围。 |
| 单任务必须可提交 | 完成后默认构建/检查、更新审核记录、提交 Git，并尝试推送 GitHub。 |
| 不混入无关改动 | 如果工作区已有其它改动，本任务提交只包含当前任务相关文件。 |
| 先基础后功能 | 优先完成数据结构、公共组件、保存回显，再做 AI 生成、多媒体任务和高级体验。 |
| 先模拟后真实模型 | 页面与数据链路先用本地模拟跑通，真实模型和平台接口后续替换。 |

## 状态标记

| 状态 | 含义 |
| --- | --- |
| `TODO` | 尚未开始 |
| `DOING` | 当前正在做 |
| `DONE` | 已完成并提交 |
| `BLOCKED` | 受外部依赖阻塞 |
| `REVIEW` | 已实现，等待复审或用户确认 |
