# coMGan-ai 主要功能页面任务状态

本文档用于跟踪《主要功能页面小任务拆分清单.md》中的任务执行状态。后续每次只完成一个小任务时，都应更新对应任务状态、提交号、验证结果和备注。

## 状态说明

| 状态 | 含义 |
| --- | --- |
| TODO | 尚未开始 |
| DOING | 当前正在做 |
| REVIEW | 已实现，等待复审或用户确认 |
| DONE | 已完成、已验证、已提交 |
| BLOCKED | 受外部依赖阻塞 |

## 后续更新规范

| 场景 | 更新方式 |
| --- | --- |
| 开始单个任务 | 将状态改为 DOING，备注写明开始时间或范围 |
| 完成实现 | 将状态改为 REVIEW，写明交付物和验证方式 |
| 提交后 | 将状态改为 DONE，并写入提交号 |
| 推送失败 | 状态仍可为 DONE，但备注必须记录 GitHub 推送失败原因 |
| 遇到阻塞 | 将状态改为 BLOCKED，并写明阻塞依赖 |
