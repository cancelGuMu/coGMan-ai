# coMGan-ai 主要功能页面开发明细表

本文档用于把《项目步骤功能说明.md》中的 11 步创作流程拆解成可开发、可验收、可排期的主要功能页面明细。后续 coMGan-ai 的创作中心页面、接口、数据结构和测试用例，默认以本文档为施工蓝图。

## 0. 页面开发总原则

| 项目 | 明细 |
| --- | --- |
| 产品目标 | 让用户从一个故事想法开始，逐步完成故事架构、剧本、资产、分镜、提示词、画面、质检、视频、声音、剪辑、发布复盘的完整 AI 漫剧生产闭环。 |
| 页面形态 | 创作中心采用 11 个步骤页面，每个步骤有独立路由、独立保存区、独立生成区、独立结果区，并能读取前序步骤产物。 |
| 公共页面骨架 | 顶部项目名与切换项目、当前进度、最近修改时间、步骤导航、状态提示、保存状态、导出入口、AI 任务队列、版本记录。 |
| 公共交互 | 新建项目、切换项目、保存草稿、AI 生成、重新生成、局部改写、导入文件、导出结果、复制内容、版本回滚、进入下一步。 |
| 公共数据要求 | 所有步骤数据必须绑定 `projectId`；每次保存记录 `updated_at`、`updated_by`、`version_status`；生成结果需要可回显、可追踪、可继续编辑。 |
| 公共验收要求 | 刷新页面不丢数据；关闭再打开可回显；每个按钮有响应；失败有提示；步骤间能读取依赖数据；移动端可读可点。 |

## 2. 全局页面组件明细
