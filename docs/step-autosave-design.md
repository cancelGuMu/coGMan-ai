# 步骤自动保存设计稿

## 覆盖任务

- `G-017` 步骤数据脏状态检测
- `G-018` 步骤自动保存设计稿

## 当前落地

步骤一已接入脏状态提示：当用户修改项目基础信息、核心故事、世界观、主线目标、人物关系或单集大纲后，页面显示“当前步骤有未保存修改”。保存成功后恢复为“当前步骤已与项目数据同步”。

## 后续自动保存规则

| 场景 | 触发策略 | UI 反馈 |
| --- | --- | --- |
| 文本输入停止 | 停止输入 1200ms 后保存草稿 | 保存中、已保存、保存失败 |
| 切换步骤 | 若存在未保存修改，先保存再跳转 | 顶部状态提示 |
| 关闭页面 | 若存在未保存修改，浏览器确认提示 | 原生离开确认 |
| AI 生成完成 | 生成结果写入后立即标记 dirty，不自动覆盖历史版本 | 生成完成但待保存 |
| 保存失败 | 保留本地输入，不清空 dirty 状态 | 显示错误并允许重试 |

## 技术建议

后续可在每个步骤组件内封装 `useStepDraftState`，统一输出 `isDirty`、`saveStatus`、`markDirty`、`markSaved` 和 `markError`，并复用 `SaveStatusBadge`。
