# UI-DOC-001：创作中心公共布局组件接入方案

## 任务边界

本文档整理 coMGan-ai 创作中心公共布局组件的接入方案，只写文档，不修改生产代码。

目标是为后续 11 个步骤页面统一接入双栏布局、三栏布局、顶部项目栏、步骤页头、保存状态、空状态和错误提示提供施工顺序。

## 接入目标

创作中心需要形成统一页面骨架：

- 顶部项目栏统一显示项目名、状态、当前进度、最近修改时间、切换项目入口。
- 步骤页头统一显示步骤编号、步骤名称、说明、上游依赖、下游去向和保存状态。
- 双栏布局承载大多数编辑页：左侧输入/编辑，右侧生成结果/预览/版本。
- 三栏布局承载复杂生产页：左侧导航/筛选，中间主编辑，右侧详情/任务/版本。
- 保存状态、空状态、错误提示统一组件化。
- 11 个步骤逐步接入，不一次性大改所有页面。

## 公共组件职责建议

### 顶部项目栏

建议组件名：`ProjectTopBar`。

| 内容 | 数据来源 | 行为 |
| --- | --- | --- |
| 项目名 | `project.name` | 支持进入重命名或项目详情 |
| 项目状态 | `project.status` | 显示草稿中、制作中、已完成等 |
| 当前进度 | `project.progress` | 显示进度条或百分比 |
| 当前步骤 | `project.current_step` | 与步骤导航保持一致 |
| 最近修改 | `project.updated_at` | 保存后刷新 |
| 切换项目 | 项目列表 | 返回项目中心或打开项目选择 |

接入要求：

- 所有步骤页面顶部一致。
- 不在每个步骤里重复写项目顶部逻辑。
- 加载失败时顶部仍能显示错误提示区域。

### 步骤页头

建议组件名：`StepPageHeader` 或复用已有 `SectionHeader`。

| 内容 | 数据来源 | 行为 |
| --- | --- | --- |
| 步骤编号 | 11 步元数据 | 显示 01 到 11 |
| 步骤名称 | 11 步元数据 | 如视频生成、发布复盘 |
| 步骤说明 | 11 步元数据 | 简短说明当前步骤目标 |
| 上游依赖 | 步骤配置 | 展示当前步骤读取哪些上游产物 |
| 下游去向 | 步骤配置 | 展示保存后服务哪些后续步骤 |
| 保存状态 | 当前步骤保存状态 | 保存中、已保存、失败 |

接入要求：

- 不写长篇说明文案挤占工作区。
- 保存状态要靠近页面标题，用户能立即看到。
- 步骤完成/阻塞状态要清晰。

### 双栏布局

建议组件名：`TwoColumnEditorLayout`。

适用步骤：

| 步骤 | 适用原因 |
| --- | --- |
| 01 故事架构 | 左侧输入故事设定，右侧生成大纲和检查结果 |
| 02 剧本创作 | 左侧素材和剧本编辑，右侧 AI 改写、审核和版本 |
| 03 资产设定 | 左侧资产卡编辑，右侧参考图、风格板和模板 |
| 05 提词生成 | 左侧镜头和提示词编辑，右侧参数、负面词、版本 |
| 11 发布复盘 | 左侧发布记录和指标，右侧复盘报告和回流建议 |

布局建议：

- 桌面端：左 60%，右 40%。
- 中等屏：保持两栏但压缩右侧。
- 移动端：上下堆叠，操作按钮固定在对应区域内。
- 两栏之间不使用嵌套卡片，使用清晰分区和边界。

### 三栏布局

建议组件名：`ThreePaneWorkbenchLayout`。

适用步骤：

| 步骤 | 适用原因 |
| --- | --- |
| 04 分镜规划 | 左侧镜头列表，中间镜头编辑，右侧详情/节奏/版本 |
| 06 画面生成 | 左侧筛选和任务，中间候选图网格，右侧素材详情 |
| 08 视频生成 | 左侧镜头任务，中间视频候选预览，右侧参数/失败/版本 |
| 09 音频字幕 | 左侧台词/角色，中间音频字幕编辑，右侧预览/混音 |
| 10 剪辑成片 | 左侧素材轨道，中间时间线，右侧导出/检查/版本 |

布局建议：

- 左栏用于导航、筛选、列表，不承载复杂编辑。
- 中栏用于主工作区，是视觉焦点。
- 右栏用于详情、状态、版本、AI 任务。
- 小屏幕可折叠左右栏，优先保留中栏。

### 保存状态组件

建议复用 `SaveStatusBadge`。

状态建议：

| 状态 | 文案 | 行为 |
| --- | --- | --- |
| `idle` | 未保存 | 编辑后显示 |
| `saving` | 保存中 | 禁止重复提交 |
| `saved` | 已保存 | 显示最近保存时间 |
| `error` | 保存失败 | 提供重试入口或错误说明 |

接入要求：

- 保存状态应由步骤数据统一驱动。
- 保存失败不能清空用户输入。
- 切换步骤、切换项目、刷新前需要未保存提示。

### 空状态组件

建议复用 `EmptyStatePanel`。

典型空状态：

| 场景 | 文案方向 | 主要操作 |
| --- | --- | --- |
| 没有故事设定 | 引导输入核心故事 | 新增故事想法 |
| 没有剧本 | 引导导入或生成剧本 | 导入文本、生成剧本 |
| 没有资产 | 引导从剧本提取资产 | 一键提取 |
| 没有镜头 | 引导拆分分镜 | 自动拆镜 |
| 没有提示词 | 引导从镜头生成提示词 | 批量生成 |
| 没有素材 | 引导创建生成任务 | 开始生成 |
| 没有复盘数据 | 引导添加发布记录 | 新增发布记录 |

接入要求：

- 空状态要提供下一步动作，而不是只写“暂无数据”。
- 空状态不替代错误状态。

### 错误提示组件

建议复用 `ErrorNotice`。

错误类型：

| 类型 | 展示方式 |
| --- | --- |
| 项目不存在 | 页面级错误，提供返回项目中心 |
| 步骤数据读取失败 | 步骤级错误，提供重试 |
| 保存失败 | 表单上方或保存区错误，保留输入 |
| 生成任务失败 | 任务卡内错误，提供重试/查看原因 |
| 上游依赖缺失 | 阻塞提示，提供跳转到上游步骤 |

接入要求：

- 错误提示必须可读、可定位、可恢复。
- 不用浏览器 alert 作为主要错误反馈。
- 错误不能遮挡用户已输入内容。

## 分阶段接入方案

### 阶段一：公共外壳

目标：让所有步骤页面先共享同一顶部和页头。

任务：

1. 接入 `ProjectTopBar`。
2. 接入 `StepPageHeader`。
3. 接入统一保存状态位置。
4. 统一项目不存在、步骤不存在错误页。

验收：

- 11 步页面顶部信息一致。
- 切换步骤后项目名、进度、最近修改时间不丢。
- 非法项目和非法步骤不白屏。

### 阶段二：双栏步骤接入

优先接入：01、02、03、05、11。

原因：这些页面以文本、表单、报告为主，双栏布局成本低、收益高。

验收：

- 左右栏内容职责清楚。
- 保存状态统一。
- 空状态有明确行动入口。
- 移动端上下堆叠可读可操作。

### 阶段三：三栏工作台接入

接入：04、06、07、08、09、10。

原因：这些页面包含列表、预览、详情、任务和版本，三栏更适合生产型工作台。

验收：

- 左栏列表可筛选。
- 中栏为主工作区。
- 右栏显示详情、任务、版本。
- 小屏幕可折叠左右栏。

### 阶段四：统一状态与门禁

目标：把保存状态、步骤状态、上游依赖缺失、门禁阻塞统一显示。

任务：

1. 接入步骤状态枚举。
2. 接入上游依赖缺失提示。
3. 接入下一步门禁提示。
4. 接入保存失败重试。

验收：

- 未保存、保存中、已保存、保存失败状态清晰。
- 上游缺失时能跳回对应步骤。
- 门禁阻塞不会静默失败。

## 每个步骤的接入注意事项

### 01 故事架构

- 左栏放项目基础信息、核心故事、世界观、主线目标。
- 右栏放季集结构、单集大纲、连续性检查。
- 空状态优先引导输入故事想法。

### 02 剧本创作

- 左栏放素材导入、小说正文、剧本文本。
- 右栏放局部改写、批量改写、审核意见、版本记录。
- 长文本区域需要稳定高度，避免保存状态跳动。

### 03 资产设定

- 左栏放角色卡、场景卡、道具卡。
- 右栏放参考图、风格板、一致性规则、提示词模板。
- 资产列表空状态要引导从剧本提取。

### 04 分镜规划

- 左栏放镜头列表和筛选。
- 中栏放镜头编辑。
- 右栏放构图、节奏、任务队列和版本。

### 05 提词生成

- 左栏放镜头选择器、T2I/I2V 提示词。
- 右栏放负面词、参数、锁定关键词、版本。
- 生成参数变更需要保存状态提示。

### 06 画面生成

- 左栏放任务筛选和镜头队列。
- 中栏放候选图网格。
- 右栏放素材详情、重绘、版本。

### 08 视频生成

- 左栏放视频任务和镜头筛选。
- 中栏放候选视频预览。
- 右栏放动作运镜参数、失败原因、版本记录。

### 09 音频字幕

- 左栏放台词和角色声音。
- 中栏放音频字幕编辑。
- 右栏放口型同步、音效、混音和预览。

### 10 剪辑成片

- 左栏放素材和轨道列表。
- 中栏放时间线。
- 右栏放导出版本、质量检查、封面候选。

### 11 发布复盘

- 左栏放发布素材、平台适配、发布记录、表现数据。
- 右栏放留存分析、复盘报告、反馈回流、实验方向。
- 指标卡和报告区域要区分清楚，避免像数据看板和编辑器混在一起。

## 数据与交互约定

1. 布局组件只负责结构，不负责业务保存。
2. 步骤页面负责把字段传给保存接口。
3. 保存状态组件从当前步骤状态读取，不自行推断业务完成度。
4. 空状态组件接收操作按钮，不硬编码步骤逻辑。
5. 错误组件接收错误类型、错误文案和恢复动作。
6. 版本记录组件只展示摘要，具体恢复逻辑由步骤页面处理。

## 验收清单

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| UI-001 | 顶部项目栏一致 | 11 步显示项目名、进度、最近修改时间一致 |
| UI-002 | 步骤页头一致 | 每步显示编号、名称、保存状态 |
| UI-003 | 双栏布局可用 | 01、02、03、05、11 在桌面端左右栏职责清晰 |
| UI-004 | 三栏布局可用 | 04、06、07、08、09、10 左中右职责清晰 |
| UI-005 | 移动端可读 | 双栏和三栏在窄屏不重叠、不溢出 |
| UI-006 | 空状态有效 | 每个步骤空数据时有下一步动作 |
| UI-007 | 错误提示有效 | 读取失败、保存失败、项目不存在都有恢复动作 |
| UI-008 | 保存状态稳定 | 保存中、已保存、失败状态不造成布局跳动 |
| UI-009 | 上游缺失提示 | 缺少依赖时能跳回对应步骤 |
| UI-010 | 不影响业务数据 | 接入布局后保存和回显行为不变 |

## 后续落地建议

1. 先做公共外壳，不急着重排所有表单。
2. 双栏步骤先接入，因为冲突风险较低。
3. 三栏步骤按 04、06、07、08、09、10 顺序逐步接入。
4. 每接入一个步骤都跑保存、刷新回显、错误提示和移动端抽查。
5. 高冲突文件由主控统一改，执行对话只提供文档方案。
