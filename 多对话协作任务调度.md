# coMGan-ai 多对话协作任务调度

本文档用于把后续功能页开发拆给多个 Codex 对话并避免互相改代码导致冲突。所有对话必须先阅读本文件，再开始执行自己的任务。

## 总控规则

1. 本对话作为主控对话，负责分配任务、审核边界、合并分支、处理冲突和最终推送 `main`。
2. 其他对话默认不能直接在 `main` 上提交，应创建自己的分支，例如 `codex/worker-a-g011-layout`。
3. 每个对话一次只做一个任务编号，除非主控明确追加任务。
4. 每个对话只能修改自己任务授权的文件；遇到未授权文件必须先停止并回报。
5. 高冲突文件由主控统一改动，其他对话默认禁止修改：
   - `apps/web/src/components.tsx`
   - `apps/web/src/components.css`
   - `apps/web/src/data.ts`
   - `apps/web/src/types.ts`
   - `apps/web/src/App.tsx`
   - `主要功能页面任务状态.md`
   - `review/agent-review-log.md`
6. 如任务必须改高冲突文件，先做设计文档或新文件，不直接改原文件。
7. 每个对话提交前必须运行 `npm run build:web` 和 `git diff --check`。
8. 每个对话提交前必须检查不能提交敏感或本地文件：
   - `项目核心记忆-coMGan-ai.md`
   - `.env*`
   - `apps/api/data/*.json`
   - `apps/api/.venv/`
   - `node_modules/`
   - `apps/web/dist/`
   - `*.tsbuildinfo`
9. 每个对话只提交自己的任务文件，不混入其他改动。
10. 推送失败时保留本地提交并回报错误，不要重置、不强推、不覆盖。

```text
6. 最终回复写清楚：任务编号、修改文件、验证结果、提交号、分支名。

你的任务包是：<在这里填 对话 A / 对话 B / 对话 C 的任务包>
```

## 主控合并节奏

1. 三个对话都推送分支后，主控逐个拉取并查看 diff。
2. 文档任务先合并，因为不碰生产代码，冲突风险低。
3. 主控再根据三个文档结果执行 G-011、G-012、G-013 等公共组件任务。
4. 每次合并后主控运行 `npm run build:web`、`git diff --check`，再推送 `main`。
5. 状态表与审核日志由主控统一补写，避免多人同时追加造成冲突。
